<Warning>
  **Песочница безопасности**: библиотека `luajava` полностью отключена. Прямой доступ к методам Java-рефлексии (таким как `getClass`, `getClassLoader`, `loadClass`, `forName` и т.д.) заблокирован на всех переданных Java-объектах в целях безопасности среды клиента.
</Warning>

## `client` (Системное управление и ресурсы)

Модуль `client` отвечает за регистрацию пользовательских модулей и загрузку текстур или файлов.

### Методы

<AccordionGroup>
  <Accordion title="client:register_module(name, description, category)" defaultOpen>
    Регистрирует новый модуль клиента.

    - **Параметры**:
      - `name` *(string)*: Название модуля в настройках клиента.
      - `description` *(string)*: Краткое описание функционала модуля.
      - `category` *(string)*: Категория (`"VISUALS"`, `"HUD"`, `"TOOLS"`).
    - **Возвращает**: Объект `LuaModule`.

    <Note>
      Вызывает ошибку, если имя модуля уже зарегистрировано встроенным плагином или другим активным скриптом.
    </Note>

    <CodeGroup>
      ```lua Регистрация модуля
      local my_module = client:register_module("My Module", "Custom Lua script", "HUD")
      ```
    </CodeGroup>
  </Accordion>

  <Accordion title="client:load_texture(textureName, filename)">
    Загружает локальную текстуру (PNG, JPG, SVG) в память.

    - **Параметры**:
      - `textureName` *(string)*: Идентификатор текстуры.
      - `filename` *(string)*: Относительный путь к файлу в директории скрипта.
    - **Возвращает**: `boolean` (`true` при успехе).

    <CodeGroup>
      ```lua Загрузка локальной текстуры
      local success = client:load_texture("custom_logo", "assets/logo.png")
      ```
    </CodeGroup>
  </Accordion>

  <Accordion title="client:load_texture_url(textureName, url)">
    Загружает текстуру напрямую по HTTP/HTTPS URL и кэширует ее локально на диске.

    - **Параметры**:
      - `textureName` *(string)*: Уникальный идентификатор текстуры.
      - `url` *(string)*: Прямая ссылка на изображение.
    - **Возвращает**: `boolean` (`true` при успехе).

    <CodeGroup>
      ```lua Загрузка по URL
      client:load_texture_url("avatar", "https://example.com/avatar.png")
      ```
    </CodeGroup>
  </Accordion>

  <Accordion title="client:download_file(url, localFilename)">
    Скачивает удаленный файл в папку скриптов.

    - **Параметры**:
      - `url` *(string)*: Ссылка на файл.
      - `localFilename` *(string)*: Путь для сохранения относительно директории скриптов.
    - **Возвращает**: `boolean` (`true` при успехе).

    <CodeGroup>
      ```lua Скачивание файла
      client:download_file("https://example.com/config.json", "config.json")
      ```
    </CodeGroup>
  </Accordion>
</AccordionGroup>

### `LuaModule` & Settings API

Объект `LuaModule` возвращается при регистрации модуля через `client:register_module()`. Он позволяет настраивать параметры модуля и подписываться на игровые события рендеринга.

#### Методы `LuaModule`

<AccordionGroup>
  <Accordion title="module:add_slider(name, description, min, max, default, step)">
    Добавляет ползунок (слайдер) настроек в меню модуля.

    - **Параметры**:
      - `name` *(string)*: Название настройки.
      - `description` *(string)*: Описание настройки.
      - `min` *(float)*: Минимальное значение ползунка.
      - `max` *(float)*: Максимальное значение ползунка.
      - `default` *(float)*: Начальное значение.
      - `step` *(float)*: Шаг изменения значения.
    - **Возвращает**: Объект `SliderSetting`.

    ```lua
    local speed_slider = module:add_slider("Speed", "Скорость вращения", 0.1, 10.0, 2.5, 0.1)
    ```
  </Accordion>

  <Accordion title="module:add_checkbox(name, description, default)">
    Добавляет переключатель (флажок) настроек в меню модуля.

    - **Параметры**:
      - `name` *(string)*: Название настройки.
      - `description` *(string)*: Описание настройки.
      - `default` *(boolean)*: Начальное состояние (`true` / `false`).
    - **Возвращает**: Объект `CheckboxSetting`.

    ```lua
    local use_blur = module:add_checkbox("Blur", "Размывать фон подложки", true)
    ```
  </Accordion>

  <Accordion title="module:add_mode(name, description, optionsTable, default)">
    Добавляет переключатель режимов (выпадающий список) настроек.

    - **Параметры**:
      - `name` *(string)*: Название настройки.
      - `description` *(string)*: Описание настройки.
      - `optionsTable` *(table)*: Массив строк с названиями режимов.
      - `default` *(string)*: Режим по умолчанию.
    - **Возвращает**: Объект `RadioSetting`.

    ```lua
    local mode = module:add_mode("ESP Mode", "Режим подсветки", {"Box", "Outline", "Chams"}, "Box")
    ```
  </Accordion>

  <Accordion title="module:add_color(name, description, defaultRgba)">
    Добавляет настройку выбора цвета (палитру).

    - **Параметры**:
      - `name` *(string)*: Название настройки.
      - `description` *(string)*: Описание настройки.
      - `defaultRgba` *(int)*: Packed RGBA цвет по умолчанию.
    - **Возвращает**: Объект `ColorSetting`.

    ```lua
    local color = module:add_color("ESP Color", "Цвет подсветки", renderer:rgba(255, 0, 0, 255))
    ```
  </Accordion>

  <Accordion title="module:add_bind(name, description, defaultKeyCode)">
    Добавляет настройку привязки клавиши (биндов).

    - **Параметры**:
      - `name` *(string)*: Название настройки.
      - `description` *(string)*: Описание настройки.
      - `defaultKeyCode` *(int)*: Код клавиши по умолчанию (из LWJGL).
    - **Возвращает**: Объект `BindSetting`.

    ```lua
    local bind = module:add_bind("Toggle Key", "Клавиша активации", 57) -- 57 = SPACE
    ```
  </Accordion>

  <Accordion title="module:add_input(name, description, default)">
    Добавляет поле текстового ввода настроек.

    - **Параметры**:
      - `name` *(string)*: Название настройки.
      - `description` *(string)*: Описание настройки.
      - `default` *(string)*: Значение по умолчанию.
    - **Возвращает**: Объект `InputSetting`.

    ```lua
    local text_input = module:add_input("Custom Text", "Текст для вывода", "Floral Dev")
    ```
  </Accordion>

  <Accordion title="module:setOnEnable(function)">
    Регистрирует callback-функцию, срабатывающую при активации (включении) модуля.

    ```lua
    module:setOnEnable(function()
        client:load_texture_url("custom_head", "https://example.com/head.png")
    end)
    ```
  </Accordion>

  <Accordion title="module:on_render(function)">
    Регистрирует callback-функцию для 2D-рендеринга (интерфейса/HUD).

    ```lua
    module:on_render(function()
        renderer:draw_text("Hello World", 10, 10, renderer:rgb(255, 255, 255), 1.0)
    end)
    ```
  </Accordion>

  <Accordion title="module:on_world_render(function(event))">
    Регистрирует callback-функцию для 3D-рендеринга в игровом мире.

    - **Параметры функции**:
      - `event` *(RenderEvent)*: Объект события 3D рендеринга.

    ```lua
    module:on_world_render(function(event)
        local wr = event:worldRenderer()
        local depth = event:frameDepth()
        -- код отрисовки 3D эффектов
    end)
    ```
  </Accordion>

  <Accordion title="module:open_settings(x, y, width)">
    Открывает нативное HUD-меню настроек данного модуля на экране.

    - **Параметры**:
      - `x` *(float)*: Координата X для появления окна.
      - `y` *(float)*: Координата Y для появления окна.
      - `width` *(float)*: Ширина окна настроек.

    ```lua
    module:open_settings(x, y + 45, 220)
    ```
  </Accordion>

  <Accordion title="module:close_settings()">
    Закрывает активное HUD-меню настроек модуля.

    ```lua
    module:close_settings()
    ```
  </Accordion>

  <Accordion title="module:isEnabled()">
    Проверяет, активен (включен) ли модуль в данный момент.

    - **Возвращает**: `boolean` (`true` если включен).

    ```lua
    local active = module:isEnabled()
    ```
  </Accordion>

  <Accordion title="module:setEnabled(enabled)">
    Программно включает или выключает модуль.

    - **Параметры**:
      - `enabled` *(boolean)*: `true` для включения, `false` для выключения.

    ```lua
    module:setEnabled(true)
    ```
  </Accordion>

  <Accordion title="module:toggle()">
    Переключает состояние модуля (включает, если выключен, и наоборот).

    ```lua
    module:toggle()
    ```
  </Accordion>
</AccordionGroup>

#### Методы настроек

| Объект | Метод | Возвращает | Описание |
| :--- | :--- | :--- | :--- |
| **Все настройки** | `setting:setValue(value)` | `void` | Программно устанавливает новое значение настройки. |
| `SliderSetting` | `setting:getValue()` | `float` | Возвращает текущее значение ползунка. |
| `CheckboxSetting` | `setting:getValue()` | `boolean` | Возвращает текущее состояние переключателя. |
| `RadioSetting` | `setting:getValue()` | `string` | Возвращает название выбранного режима. |
| `InputSetting` | `setting:getValue()` | `string` | Возвращает введенную строку текста. |
| `BindSetting` | `setting:getKeyCode()` | `int` | Возвращает текущий код привязанной клавиши. |
| `BindSetting` | `setting:getKeyName()` | `string` | Возвращает название привязанной клавиши. |
| `ColorSetting` | `setting:getValue():toRgba()` | `int` | Возвращает текущий цвет в формате Packed RGBA. |
| `ColorSetting` | `setting:getValue():hue()` | `float` | Hue (оттенок) цвета. |
| `ColorSetting` | `setting:getValue():saturation()` | `float` | Saturation (насыщенность) цвета. |
| `ColorSetting` | `setting:getValue():brightness()` | `float` | Brightness (яркость) цвета. |
| `ColorSetting` | `setting:getValue():alpha()` | `float` | Alpha (прозрачность) цвета. |

#### Событие `RenderEvent`

Передается в функцию `on_world_render`.

| Метод | Возвращает | Описание |
| :--- | :--- | :--- |
| `event:worldRenderer()` | `WorldRenderer` | Возвращает объект 3D-рендерера мира для отрисовки геометрии. |
| `event:frameDepth()` | `float` | Возвращает время промежуточного кадра (partial ticks) для плавной интерполяции позиций. |

## `renderer` (2D & 3D рендеринг)

Модуль `renderer` предоставляет функции отрисовки элементов интерфейса, текста, эффектов размытия и 3D-объектов в мире.

### Цвета и шейдеры

| Метод | Возвращает | Описание |
| :--- | :--- | :--- |
| `renderer:rgb(r, g, b)` | `int` | Создать RGB цвет (значения 0-255). |
| `renderer:rgba(r, g, b, a)` | `int` | Создать RGBA цвет с альфа-каналом. |
| `renderer:get_client_primary_color()` | `int` | Получить основной цвет темы клиента. |
| `renderer:get_client_primary_color_alpha(alpha)` | `int` | Получить основной цвет темы с прозрачностью. |
| `renderer:prepare_blur(strength)` | `void` | Подготовить буфер размытия с заданной силой. |

### 2D Отрисовка и текст

| Метод | Описание |
| :--- | :--- |
| `renderer:draw_rect(x, y, w, h, color)` | Нарисовать заполненный прямоугольник. |
| `renderer:draw_rect_rounding(x, y, w, h, rounding, color)` | Прямоугольник с закругленными углами. |
| `renderer:draw_gradient(x, y, w, h, colorTL, colorTR, colorBR, colorBL)` | Прямоугольник с градиентом по вершинам (Packed RGBA цвета). |
| `renderer:draw_circle(cx, cy, radius, startDeg, pct, color)` | Заполненный круг или сектор (углы в градусах, `pct` от 0.0 до 1.0). |
| `renderer:draw_circle_outline(cx, cy, radius, startDeg, pct, thickness, color)` | Контур круга или дуга заданной толщины. |
| `renderer:draw_shadow(x, y, w, h, rounding, blurStrength, spread, color)` | Мягкая внешняя тень для прямоугольных элементов интерфейса. |
| `renderer:draw_blur(x, y, w, h, rounding)` | Область размытия с закруглением. |
| `renderer:draw_blur_alpha(x, y, w, h, rounding, alpha)` | Область размытия с прозрачностью. |
| `renderer:draw_text(text, x, y, color, size)` | Отрисовка текста по координатам. |
| `renderer:draw_text_center(text, x, y, color, size)` | Отрисовать текст с выравниванием по центру координат `(x, y)`. |
| `renderer:draw_text_centered_in_rect(text, x, y, w, h, color, size)` | Текст по центру заданной области. |
| `renderer:get_text_width(text, size)` → `float` | Рассчитать ширину текста. |
| `renderer:get_centered_baseline_offset(size)` → `float` | Смещение базовой линии шрифта. |
| `renderer:draw_texture(textureName, x, y, w, h, color)` | Отрисовка текстуры. |
| `renderer:draw_texture_centered_in_rect(textureName, rx, ry, rw, rh, destW, destH, color)` | Отрисовка текстуры по центру прямоугольной области. |
| `renderer:draw_texture_rotated(textureName, x, y, w, h, rotationDegrees, color)` | Отрисовка повернутой текстуры. |
| `renderer:draw_texture_centered_in_rect_rotated(textureName, rx, ry, rw, rh, destW, destH, rotationDegrees, color)` | Повернутая текстура по центру прямоугольной области. |
| `renderer:draw_texture_region(textureName, x, y, w, h, u0, v0, u1, v1, color)` | Отрисовка части (региона) текстуры по UV-координатам (0.0 - 1.0). |
| `renderer:draw_texture_region_rounded(textureName, x, y, w, h, u0, v0, u1, v1, rounding, color)` | Отрисовка кадрированной текстуры с закруглением. |
| `renderer:draw_texture_region_rotated(textureName, x, y, w, h, u0, v0, u1, v1, rotationDegrees, color)` | Отрисовка повернутой части (региона) текстуры. |

### Трансформации и перетаскивание

| Метод | Возвращает | Описание |
| :--- | :--- | :--- |
| `renderer:begin_drag(id, preferredX, preferredY, w, h)` | `DragSession` | Начать сессию перетаскивания элемента. |
| `renderer:end_drag(session)` | `void` | Завершить сессию перетаскивания. |
| `renderer:render_drag_outline(session, cornerRadius, visibilityProgress)` | `void` | Нарисовать рамку вокруг перетаскиваемого HUD-элемента. |
| `renderer:push_scale(scaleX, scaleY, centerX, centerY)` | `void` | Применить масштабирование относительно центра. |
| `renderer:pop_scale()` | `void` | Сбросить масштабирование. |

#### Сессия перетаскивания `DragSession`

Возвращается методом `renderer:begin_drag()`. Позволяет реализовать интерактивное перетаскивание HUD-элементов мышкой по экрану.

| Метод | Возвращает | Описание |
| :--- | :--- | :--- |
| `session:id()` | `string` | Возвращает строковый идентификатор сессии перетаскивания. |
| `session:positionX()` | `float` | Возвращает текущую координату X перетаскиваемого элемента (или разрешенную предпочтительную координату X). |
| `session:positionY()` | `float` | Возвращает текущую координату Y перетаскиваемого элемента (или разрешенную предпочтительную координату Y). |
| `session:width()` | `float` | Возвращает ширину области перетаскивания. |
| `session:height()` | `float` | Возвращает высоту области перетаскивания. |
| `session:dragging()` | `boolean` | Возвращает `true`, если элемент перетаскивается в данный момент. |
| `session:alphaAdjusted()` | `boolean` | Возвращает `true`, если прозрачность элемента была скорректирована менеджером перетаскивания. |
| `session:rightClicked()` | `boolean` | Возвращает `true`, если по элементу был совершен правый клик в этом кадре. |

### 3D Рендеринг

| Метод | Возвращает | Описание |
| :--- | :--- | :--- |
| `renderer:get_camera_vectors(worldRenderer)` | `float[]` | Получить векторы камеры `[rxX, rxY, rxZ, ruX, ruY, ruZ]`. |
| `renderer:draw_quad_3d(worldRenderer, x1,y1,z1, x2,y2,z2, x3,y3,z3, x4,y4,z4, color, disableDepth)` | `void` | Отрисовка залитого 3D-полигона (4 вершины в мире) без текстуры. |
| `renderer:draw_textured_quad_3d(worldRenderer, textureName, x1,y1,z1, x2,y2,z2, x3,y3,z3, x4,y4,z4, u0,v0, u1,v1, u2,v2, u3,v3, color, isAdditive, disableDepth)` | `void` | Отрисовка 3D текстурированного квадрата в мире. |
| `renderer:draw_text_3d(worldRenderer, text, x, y, z, color, scale, disableDepth)` | `void` | Отрисовка 3D-текста (биллборда) в мире по указанным координатам. |
| `renderer:draw_cube(worldRenderer, targetLuaEntity, animValue, redIntensity, textureType, baseColor, redOnHit)` | `void` | Отрисовка детализированного куба (повороты, эффекты попадания) над сущностью. |
| `renderer:draw_cube_simple(worldRenderer, targetLuaEntity, tickDelta, size, color, useSecondTexture)` | `void` | Отрисовка простого куба над сущностью с текстурой. |
| `renderer:draw_target_esp(worldRenderer, targetLuaEntity, mode, frameDepth, movingValue, alpha, clientColor, accentColor, redOnHit, impactProgress)` | `void` | Отрисовка нативного эффекта Target ESP клиента вокруг цели. |
| `renderer:update_target_esp()` | `void` | Обновить состояние Target ESP. |

<CodeGroup>
  ```lua Пример рендеринга с блюром
  function on_render()
      local color_bg = renderer:rgba(15, 15, 20, 180)
      local color_text = renderer:get_client_primary_color()

      renderer:prepare_blur(8.0)
      renderer:draw_blur(50, 50, 200, 100, 12)
      renderer:draw_rect_rounding(50, 50, 200, 100, 12, color_bg)
      renderer:draw_text_centered_in_rect("Floral", 50, 50, 200, 100, color_text, 1.2)
  end
  ```
</CodeGroup>

## `minecraft` (Состояние игры и данные мира)

Доступ к состоянию игры, параметрам сущностей, клавишам управления и эффектам.

### Основное состояние игры

| Метод | Тип ответа | Описание |
| :--- | :--- | :--- |
| `minecraft:is_in_world()` | `boolean` | Находится ли игрок в мире/на сервере. |
| `minecraft:is_chat_open()` | `boolean` | Открыт ли игровой чат. |
| `minecraft:get_fps()` | `int` | Текущий FPS. |

## 3. Практический пример: Перетаскивание и нативное меню настроек по правому клику

Ниже представлен полноценный рабочий пример создания пользовательского HUD-модуля, который поддерживает перетаскивание мышкой, масштабирование и открывает нативное меню настроек Floral при клике правой кнопкой мыши:

```lua drag_hud.lua
-- Регистрируем наш модуль в категории HUD (он появится в оверлей-меню Floral)
local module = client:register_module("Lua Drag HUD", "Пример HUD с меню настроек по правому клику", "HUD")

-- Оставляем только нужные настройки, позицию X и Y клиент сохранит автоматически по ID сессии
local scale_setting = module:add_slider("HUD Scale", "Размер шрифта", 8, 20, 11, 1)

-- Вспомогательные переменные для оптимизации (чтобы не перегружать ПК вычислениями каждый кадр)
local cached_username = ""
local cached_text = ""
local cached_width = 120
local last_update_time = 0.0

-- Главный цикл рендеринга модуля (вызывается каждый игровой кадр)
module:on_render(function()
    if not minecraft:is_in_world() then return end

    local current_time = os.clock()
    local scale = scale_setting:getValue()

    -- 1. Считываем имя игрока только один раз на старте
    if cached_username == "" then
        cached_username = minecraft:get_username()
    end

    -- 2. Обновляем FPS и ширину текста по таймеру раз в 150мс (вместо каждого кадра)
    if current_time - last_update_time > 0.15 or cached_text == "" then
        cached_text = "FPS: " .. tostring(minecraft:get_fps()) .. "  |  User: " .. cached_username
        cached_width = renderer:get_text_width(cached_text, scale)
        last_update_time = current_time
    end

    -- 3. Рассчитываем размеры рамки с учетом отступов
    local width = cached_width + 24
    local height = scale + 18

    -- 4. Получаем координаты перетаскивания (100, 100 — это позиция при самом первом запуске)
    local session = renderer:begin_drag("floral.custom_fps_hud", 100, 100, width, height)
    local x = session:positionX()
    local y = session:positionY()

    -- Открываем настройки модуля при клике правой кнопкой мыши по HUD
    if session:rightClicked() then
        module:open_settings(x, y + height + 6, 200)
    end

    -- Отрисовываем контур HUD во время его перемещения в режиме кастомизации
    if session:dragging() then
        renderer:render_drag_outline(session, 8, 1.0)
    end

    -- 5. Рисуем фон с красивым размытием
    local color_bg = renderer:rgba(18, 18, 22, 195)
    local color_primary = renderer:get_client_primary_color()

    renderer:prepare_blur(6.0)
    renderer:draw_blur(x, y, width, height, 8)
    renderer:draw_rect_rounding(x, y, width, height, 8, color_bg)

    -- 6. Отрисовываем текст ровно по центру рамки
    renderer:draw_text_centered_in_rect(cached_text, x, y, width, height, color_primary, scale)

    -- 7. Завершаем сессию перетаскивания
    renderer:end_drag(session)
end)
```
